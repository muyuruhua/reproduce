# 漏洞复现操作指南（审核人员用）

---

## 方式一：从 Excel 精确复现（推荐）

审核员打开 `vulnerability/` 里的 Excel 报告 → 看到一条漏洞 → 一条命令复现。

```bash
# Excel 中看到 bftpd, Category=0x0020 的 CRLF 注入漏洞
bash scripts/reproduce_from_excel.sh bftpd viol 0x0020
```

输出中看到即确认：

```
✅ VIOLATION CONFIRMED — 逻辑漏洞已独立复现!
  [HIGH] CRLF_INJECTION: ... (CWE-93, CVE-2026-39983)
```

```bash
# 崩溃复现 (持久重放模式: 启动一次, 反复重放, 堆状态逐步累积)
bash scripts/reproduce_from_excel.sh bftpd crash
# 输出: ✅ CRASH REPRODUCED — 内存破坏漏洞已独立复现!

# 也可指定目标 (可复现: bftpd/proftpd/live555)
bash scripts/reproduce_from_excel.sh proftpd crash
bash scripts/reproduce_from_excel.sh live555 crash
```

**Category 速查**：Excel 中 `Category=0x0020` → 命令 `viol 0x0020` 即可。详见 `漏洞复现演示.md` 中全部 10 个目标的 Category↔命令对应表。

## 方式二：一键批量复现

```bash
cd reproduce

# 1. 导入镜像（首次，约3分钟）
for f in docker_images/*.tar.gz; do docker load -i "$f"; done

# 2. 一键复现全部 10 个目标（约8分钟）
bash quickstart.sh --demo

# 3. 查看结果
grep -a "CRASH DETECTED\|Confirmed" /tmp/chatafl_reproduce_*/
```

## 手动复现单个种子（可选）

```bash
# 用预提取的样本种子
bash scripts/replay_crash.sh bftpd bftpd/seeds/crashes/crash_1_*

# 或从原始 tarball 自己提取
tar xzf ten_groups_data_ten/results-bftpd_Mar-16_23-10-02_ten/out-bftpd-chatafl_opt_1.tar.gz \
    "out-bftpd-chatafl_opt/replayable-crashes/id:000009,sig:06,..."
bash scripts/replay_crash.sh bftpd out-bftpd-chatafl_opt/replayable-crashes/id:000009,sig:06,...
```
