# 漏洞复现操作指南（审核人员用）

**只需 3 条命令，约 10 分钟完成全部 10 个目标的漏洞复现验证。**

---

## 步骤 1：导入 Docker 镜像（2-3 分钟）

```bash
cd reproduce

# 导入所有镜像
for f in docker_images/*.tar.gz; do
    docker load -i "$f"
done

# 验证（应显示全部 10 个 OK）
for img in lightftp bftpd proftpd pure-ftpd exim live555 kamailio forked-daapd lighttpd1 mosquitto-v2.0.18; do
    docker image inspect "$img:latest" >/dev/null 2>&1 && echo "OK  $img" || echo "MISS $img"
done
```

## 步骤 2：一键复现（5-8 分钟）

```bash
# 快速演示模式 — 每个目标只测 1 个种子，约 5 分钟
bash quickstart.sh --demo

# 或完整模式 — 每个目标测试全部样本种子，约 8 分钟
bash quickstart.sh
```

## 手动复现单个漏洞（可选）

```bash
# 崩溃复现（以 bftpd 为例）
bash scripts/replay_crash.sh bftpd \
    bftpd/seeds/crashes/crash_1_* \
    /tmp/manual_test

cat /tmp/manual_test/replay.log | grep "CRASH DETECTED\|AddressSanitizer"
# 预期输出:
#   [CRASH DETECTED] replay #1: server crashed! exit_code=0
#   ERROR: AddressSanitizer: heap-buffer-overflow ...

# 逻辑漏洞复现（以 bftpd 为例）
bash scripts/replay_logical_vuln.sh bftpd \
    bftpd/seeds/violations/viol_2_* \

# 预期输出:

#     Severity: HIGH
#     Category: AUTH_BYPASS
```

## 故障排除

| 问题 | 解决 |
|------|------|
| `docker: permission denied` | `sudo usermod -aG docker $USER && newgrp docker` |
| 磁盘空间不足 | `docker system prune -a` 清理旧镜像 |
| 导入极慢 | 可先解压：`gunzip -c xxx.tar.gz \| docker load` |
| 端口冲突 | 脚本使用 `--network host`，确保端口 21/25/1883/3689/5060/8080/8554 空闲 |
