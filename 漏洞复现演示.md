# 漏洞复现演示

**审核人员在 Excel 报告中看到漏洞 → 在本目录找到对应的种子 → 一条命令复现。**

---

## 一、总体流程

```
打开 vulnerability/ 中的 Excel → 找到感兴趣的漏洞行
    │
    │  记下 Target(目标名) / Category(类型编号) / Severity(严重程度)
    │
    ▼
进入 <target>/seeds/ 目录 → 找文件名匹配的种子
    │
    │  类别掩码速查: 0x0001=AUTH_BYPASS  0x0004=STATE_VIOLATION
    │              0x0020=INJECTION      0x0400=SMUGGLING
    │              0x0008=INFO_LEAK      0x0010=PATH_TRAVERSAL
    │              0x0040=DOS            0x0100=ISOLATION
    │
    ▼
bash scripts/replay_crash.sh <target> <种子文件>        (崩溃)
bash scripts/replay_logical_vuln.sh <target> <种子文件>  (逻辑漏洞)
    │
    ▼
看输出 → [CRASH DETECTED] 或 [CONFIRMED] → 漏洞已独立复现
```

## 二、实战演示：从 Excel 报告到漏洞复现

以 **bftpd** 为例。审核员打开 `vulnerability/bftpd_漏洞发现_20260606145639.xlsx`，在 Vulnerability List 中看到：

| Severity | Category | CWE | Description | CVE Pattern |
|----------|----------|-----|-------------|-------------|
| 4 (HIGH) | 0x0020 | CWE-93 | CRLF injection in FTP command arguments | CVE-2026-39983 |
| 4 (HIGH) | 0x0004 | CWE-696 | RNTO without prior RNFR accepted by server | N/A |
| 5 (CRITICAL) | 0x0008 | CWE-200 | Sensitive info leaked in server response | CVE-2024-42650 |

审核员想验证 CRLF 注入漏洞。操作如下：

### 步骤 1：记下关键信息

从 Excel 行得知：`Target=bftpd`, `Category=0x0020`, `Severity=4`

### 步骤 2：在 seeds/ 中找匹配种子

```bash
ls bftpd/seeds/violations/
# viol_1_id_000005_sev_4_cat_0020  ← sev=4, cat=0020 匹配!
# viol_2_id_000006_sev_4_cat_0024
# viol_3_id_000007_sev_5_cat_0008
```

`viol_1` 的 `sev_4_cat_0020` 精确匹配 Excel 中的 Severity=4, Category=0x0020。这就是触发 CRLF 注入的种子。

### 步骤 3：一条命令复现

```bash
bash scripts/replay_logical_vuln.sh bftpd \
    bftpd/seeds/violations/viol_1_id_000005_sev_4_cat_0020
```

### 步骤 4：看到确认

```
Verifying FTP ... req=122B resp=423B
Done: 1 violation(s)
  [HIGH] CRLF_INJECTION: CRLF injection in FTP command: b'...\r\nSY?T\r\n...'
[CONFIRMED] 1 oracle violation(s) verified
```

**输出中 `[HIGH] CRLF_INJECTION` + `[CONFIRMED]` = 漏洞已独立复现。**

## 四、Category 掩码速查

| 掩码 | 名称 | 一句话解释 |
|:---:|------|------|
| 0x0001 | AUTH_BYPASS | 没登录就能干特权操作 |
| 0x0004 | STATE_VIOLATION | 跳过RFC规定的协议状态转换 |
| 0x0008 | INFO_LEAK | 响应中泄露了密码文件/配置/版本 |
| 0x0010 | PATH_TRAVERSAL | `../` 越权访问文件系统 |
| 0x0020 | INJECTION | 命令参数中含嵌入换行(CRLF)可走私命令 |
| 0x0040 | DOS | 畸形输入导致服务不可用 |
| 0x0080 | RESOURCE_EXHAUST | 畸形输入消耗完内存/连接/文件句柄 |
| 0x0100 | ISOLATION | FTP Bounce: PORT指向内网IP被接受 |
| 0x0400 | SMUGGLING | HTTP请求走私(多CL头/CL+TE共存) |

**复合 Category**（如 0x0024）表示该种子同时触发多种漏洞。数值 = 各掩码的 OR 结果。

## 五、快速操作速查

```bash
cd reproduce

# 导入镜像（首次）
for f in docker_images/*.tar.gz; do docker load -i "$f"; done

# 全自动演示（约8分钟）
```
