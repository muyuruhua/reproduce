# 从 Excel 报告到漏洞复现

审核员打开 `vulnerability/` 里的 Excel → 看到一条漏洞 → 一条命令复现。

---

## 核心关系

```
ten_groups_data_ten/                         vulnerability/
  results-bftpd_.../                           bftpd_漏洞发现_...xlsx
    out-*-chatafl_opt_1.tar.gz                 ├─ Severity: 4 (HIGH)
      replayable-violations/                   ├─ Category: 0x0020
        id:000005,sev:4,cat:0020  ──────────→  ├─ CWE: CWE-93
           ▲                                   └─ Description: CRLF injection...
           │                                            │
           │          Category 值 0x0020               │
           │          对应种子文件名 cat:0020            │
           └────────────────────────────────────────────┘
```

**`ten_groups_data_ten/` 是原始数据（290个tar.gz）。`vulnerability/` 的 Excel 由它生成。种子文件名中的 `cat:0020` 等于 Excel 的 Category 列 `0x0020`。**

---

## 操作：3 种方式

### 方式 1：一键（全自动）

```bash
# 审核员在 Excel 中看到 bftpd, Category=0x0020 的漏洞
bash scripts/reproduce_from_excel.sh bftpd viol 0x0020
```

脚本自动完成：搜索所有 tarball → 找到 cat 匹配的种子 → 提取 → Docker 重放 → 输出判定。

### 方式 2：用预提取的样本种子（<target>/seeds/）

```bash
ls bftpd/seeds/violations/
# viol_1_..._cat_0020  ← cat_0020 = Excel的0x0020

bash scripts/replay_logical_vuln.sh bftpd \
    bftpd/seeds/violations/viol_1_id_000005_sev_4_cat_0020
```

### 方式 3：从 tarball 手动提取

```bash
# 从原始数据手动提取指定种子
tar xzf ten_groups_data_ten/results-bftpd_Mar-16_23-10-02_ten/out-bftpd-chatafl_opt_1.tar.gz \
    "out-bftpd-chatafl_opt/replayable-violations/id:000005,sev:4,cat:0020"

bash scripts/replay_logical_vuln.sh bftpd \
    out-bftpd-chatafl_opt/replayable-violations/id:000005,sev:4,cat:0020
```

## 判定：看到什么算漏洞

### 崩溃 — 输出中出现这行即确认

```
[CRASH DETECTED] replay #1: server crashed! exit_code=0
  RESULT: CRASH CONFIRMED — 确认内存破坏漏洞
```

| 相关信号 | CWE | CVSS |
|---------|-----|------|
| SIGABRT(6) + ASAN报告 | CWE-122/416/415 | 7.5–9.8 |
| SIGSEGV(11) | CWE-119/125 | 7.5 |

### 逻辑漏洞 — 输出中出现这行即确认

```
[CONFIRMED] N oracle violation(s) verified
```

其上方列出的每个 `[HIGH]` / `[CRITICAL]` 行是一个被破坏的安全属性：

| 行首标记 | 含义 | CWE |
|---------|------|-----|
| `AUTH_BYPASS: ... code 200` | 未认证即可操作 | CWE-306 |
| `CRLF_INJECTION: ... ` | 命令走私 | CWE-93 |
| `FTP_BOUNCE: ...` | PORT指向内网 | CWE-441 |
| `PATH_TRAVERSAL: ...` | ../越权访问 | CWE-22 |
| `STATE_VIOLATION: ...` | RFC状态违规 | CWE-696 |
| `SMUGGLING: ...` | HTTP请求走私 | CWE-444 |
| `INFO_LEAK: ...` | 敏感信息泄露 | CWE-200 |
| `ACL_BYPASS: ...` | $SYS话题越权 | CWE-284 |
