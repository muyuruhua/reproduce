# 漏洞复现演示

**审核人员只需知道三件事：输入是什么、输出看哪里、怎么判定是漏洞。**

---

## 一、操作

```bash
cd reproduce

# 1. 导入镜像（首次，约3分钟）
for f in docker_images/*.tar.gz; do docker load -i "$f"; done

# 2. 一键复现（约8分钟）
bash quickstart.sh --demo

# 3. 查看结果
grep -r "CRASH DETECTED\|Confirmed" /tmp/chatafl_reproduce_*/
```

---

## 二、输入来源

| 你需要的 | 位置 | 格式 |
|---------|------|------|
| Docker 镜像 | `docker_images/*.tar.gz` | `docker load -i` 导入 |
| 漏洞种子 | `ten_groups_data_ten/results-*/out-*.tar.gz` | 内含 `replayable-crashes/` 和 `replayable-violations/` |
| 漏洞清单 | `vulnerability/*.xlsx` | 每条含 Severity/Category/CWE/CVE/Description |

**Excel 和 Tarball 的对应关系**：`vulnerability/bftpd_漏洞发现_...xlsx` 记录"发现了什么"，`ten_groups_data_ten/results-bftpd_.../out-*.tar.gz` 提供"用什么复现"。两者通过目标名 `bftpd` 对应。

---

## 三、输出解读

### 崩溃类（以 bftpd 为例）

执行：
```bash
bash scripts/replay_crash.sh bftpd <种子文件路径>
```

输出中**唯一需要看的一行**：

```
[CRASH DETECTED] replay #1: server crashed! exit_code=0
```

**判定**：网络服务端接收远程输入后进程崩溃 = 内存破坏漏洞。

### 逻辑漏洞类（以 bftpd 为例）

执行：
```bash
bash scripts/replay_logical_vuln.sh bftpd <种子文件路径>
```

输出中**唯一需要看的一行**：

```
[CONFIRMED] 4 oracle violation(s) verified
```

其上方列出具体破坏的安全属性，例如 `[HIGH] AUTH_BYPASS`、`[HIGH] CRLF_INJECTION`。有 `[HIGH]` 或 `[CRITICAL]` 前缀的行就是漏洞。

**判定原则**：程序不崩 ≠ 没漏洞。"程序有没有做了安全上不该做的事？" 没认证让操作 = 认证绕过；把 CRLF 当命令分隔符 = 命令走私。

## 四、判定速查表

### 崩溃类 — 看到这些关键字即确认

| 输出关键字 | CWE | CVSS |
|-----------|-----|------|
| `CRASH DETECTED` | CWE-122/416/415 | 7.5+ |
| `AddressSanitizer: heap-buffer-overflow` | CWE-122 | 9.8 |
| `AddressSanitizer: heap-use-after-free` | CWE-416 | 9.8 |
| `AddressSanitizer: stack-buffer-overflow` | CWE-121 | 9.8 |

### 逻辑漏洞类 — 看到这些关键字即确认

| 输出关键字 | 含义 | CWE | 典型 CVE |
|-----------|------|-----|---------|
| `AUTH_BYPASS` | 未认证即可执行特权操作 | CWE-306 | CVE-2024-42644 |
| `CRLF_INJECTION` | 命令参数含嵌入换行，可走私命令 | CWE-93 | CVE-2026-39983 |
| `FTP_BOUNCE` | PORT 指向内部 IP 被接受 | CWE-441 | CVE-2018-15516 |
| `PATH_TRAVERSAL` | `../` 路径越权访问 | CWE-22 | CVE-2024-3935 |
| `STATE_VIOLATION` | RFC 状态转换被跳过 | CWE-696 | N/A |
| `SMUGGLING` | 多 Content-Length / CL+TE 共存 | CWE-444 | CVE-2023-25690 |
| `INFO_LEAK` | 响应含 `/etc/passwd` 或 `root:` | CWE-200 | CVE-2024-42650 |
| `ACL_BYPASS` | 非特权客户端访问 `$SYS` 话题 | CWE-284 | CVE-2017-7650 |
| `PRIVILEGE_ESC` | Will 消息写入 `$SYS` | CWE-250 | N/A（新发现） |
| `SESSION_HIJACK` | 空 ClientID + clean_session=0 | CWE-384 | CVE-2014-6116 |
